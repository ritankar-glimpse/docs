<Info>
  **Endpoint:** `GET /api/v1/nmarket/v2/batches/{batch_id}/active-markets`\
  **Auth:** Public
</Info>

Paginated active markets feed for progressive UI loading. Returns the same shape as the v1 endpoint plus pagination metadata.

<Note>
  For the non-paginated v1 version, see [`GET /batches/{batch_id}/active-markets`](/api-reference/nmarket/active-markets).
</Note>

## Parameters

### Path

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `batch_id` | string (UUID) | yes | Batch to fetch active markets for. |

### Query

| Name | Type | Required | Description |
| --- | --- | --- | --- |
| `page` | integer | no | Page number. Defaults to `1`. |
| `page_size` | integer | no | Page size. Defaults to `10`. |

## Request

```bash
curl -X 'GET' \
  'https://main.bpmapi.io/api/v1/nmarket/v2/batches/c6dd0be7-b9b8-4a8a-b735-623660c38982/active-markets?page=1&page_size=10' \
  -H 'accept: application/json'
```

### Request URL

```text
https://main.bpmapi.io/api/v1/nmarket/v2/batches/c6dd0be7-b9b8-4a8a-b735-623660c38982/active-markets?page=1&page_size=10
```

**Top-level fields**

| Field | Type | Description |
| --- | --- | --- |
| `batch_id` | string (UUID) | Echoes the requested batch id. |
| `main_topic_title` | string | Display title for the batch. |
| `topic_count` | integer | Number of topics in the batch. |
| `start_time_utc` | integer | Batch start time, Unix seconds UTC. |
| `end_time_utc` | integer | Batch end time, Unix seconds UTC. |
| `cumulative_volume_millisats` | integer | Sum of lifetime volume across the batch, in millisats. |
| `cumulative_liquidity_millisats` | integer | Sum of locked liquidity across the batch, in millisats. |
| `count` | integer | Number of markets in this page. |
| `total` | integer | Total active markets in the batch. |
| `limit` | integer | Page size used. |
| `offset` | integer | Offset for this page. |
| `next_offset` | integer | Offset to request for the next page. |
| `has_more` | boolean | True if more pages are available. |
| `markets[]` | array | Active markets for this page. |

**Market object**

| Field | Type | Description |
| --- | --- | --- |
| `topic_id` | integer | Market identifier. |
| `title` | string | Market question / display title. |
| `description` | string | Long-form description. |
| `topic_type` | string | Topic type (e.g. `btc`). |
| `category` | string | Display category. |
| `topic_image_url` | string | URL of the topic image. |
| `batch_id` | string (UUID) | Parent batch id. |
| `end_time_utc` | integer | Market end time, Unix seconds UTC. |
| `estimated_resolve_time` | integer | Estimated resolution time, Unix seconds UTC. |
| `date_created` | integer | Creation timestamp, Unix seconds UTC. |
| `num_outcomes` | integer | Total number of outcomes defined for the market. |
| `is_active` | boolean | Whether the market is open for trading. |
| `is_resolved` | boolean | Whether the market has resolved. |
| `quote_mode` | string | Quote engine mode (e.g. `live`, `active`). |
| `market_subsidised` | boolean | Whether the market has subsidised liquidity. |
| `subsidy_amount` | integer | Subsidy amount, in millisats. |
| `resolved_option_id` | integer | `option_id` of the winning outcome (if resolved). |
| `total_amount_in_market` | integer | Total notional in the market, in millisats. |
| `volume_24h_millisats` | integer | 24-hour trading volume, in millisats. |
| `total_volume_millisats` | integer | Lifetime trading volume, in millisats. |
| `outcomes[]` | array | Slim outcome objects. |

**Outcome object** (v2 slim shape)

| Field | Type | Description |
| --- | --- | --- |
| `option_id` | integer | Outcome identifier within the market. |
| `name` | string | Display label (e.g. price bin). |
| `shares` | number | Outstanding shares for this outcome. |

<Note>
  The v2 outcome shape is intentionally slimmer than v1 — it omits `yes_price`, `no_price`, and millisats variants for faster progressive loading. Fetch [`GET /markets/{topic_id}/quotes`](/api-reference/nmarket/market-quotes) for full pricing.
</Note>

### `400 Bad Request`

```json
{
  "additionalProp1": {}
}
```

Returned when `batch_id` is missing or malformed, or pagination params are out of range.

### `500 Internal Server Error`

```json
{
  "additionalProp1": {}
}
```

Returned on unexpected server-side failures.
