---
title: "Lightning"
description: "Custodial Lightning wallets, invoices, payments, withdrawals"
---

Custodial Lightning wallet routes on the Main Service. All require **User JWT**. For top-up / pay-invoice / withdraw flows exposed by the Payments service, see [Payments](/api-reference/payments).

## Endpoints

| Method | Path | Auth | Purpose |
| --- | --- | --- | --- |
| GET | `/api/v1/lightning/wallet` | User JWT | Get Lightning wallet details. |
| POST | `/api/v1/lightning/wallet` | User JWT | Set up a custodial Lightning wallet. |
| GET | `/api/v1/lightning/wallet/balances` | User JWT | Lightning wallet balance(s). |
| POST | `/api/v1/lightning/wallet/:wallet_id/invoice` | User JWT | Create a Lightning invoice (deposit). |
| POST | `/api/v1/lightning/wallet/:wallet_id/payment` | User JWT | Pay a Lightning invoice. |
| POST | `/api/v1/lightning/wallet/:wallet_id/withdraw` | User JWT | Withdraw sats via Lightning. |
