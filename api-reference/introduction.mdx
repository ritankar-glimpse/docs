Glimpse exposes its **Main Service** over HTTP. This reference catalogs the endpoints so you can drop the full surface into an LLM context window in one pass.

## Services

| Service | Base URL |
| --- | --- |
| **Main Service** | `https://main.bpmapi.io` |

All paths below are prefixed `/api/v1/...` unless noted.

## Authentication

The service uses a **raw JWT** in the `Authorization` header — **no `Bearer ` prefix**.

```http
Authorization: <jwt>
```

Authentication tiers used throughout this reference:

- **Public** — no auth required.
- **User JWT** — valid session token in `Authorization` header.
- **User JWT \+ Eligibility** — JWT plus a passed internal trade-eligibility gate. Treated as User JWT from a caller's perspective; the gate is enforced server-side.

Obtain a JWT via [`POST /api/v1/auth/login`](/api-reference/auth) → `{ "token": "<jwt>" }`.

## Conventions

- Amounts use **sats** or **millisats** as named in the field. `1 sat = 1,000 millisats`.
- Timestamps are Unix epoch seconds, UTC.
- `topic_id` is a numeric market id. `batch_id` is a UUID grouping markets.

## ⚠ Response shape caveat

The upstream Postman collections ship with **zero saved example responses**, and the Go test suite asserts status codes only. **Only the following response shapes are verified in source:**

- `POST /api/v1/auth/login` → `{ "token": "<jwt>" }`

For every other endpoint, we document **method, path, auth, and purpose** derived from handler names and route groupings. Response bodies are **not invented** — hit a running instance or consult `internal/routes/*.go` and `internal/nmarket/api/routes.go` for exact shapes.
